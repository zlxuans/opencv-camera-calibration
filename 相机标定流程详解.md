## 1. 相机标定概述

### 1.1 什么是相机标定？

相机标定是确定相机几何参数和光学参数的过程。这些参数描述了三维世界中的点如何投影到二维图像平面上。

### 1.2 需要标定的参数

| 参数类型 | 参数名称 | 符号 | 维度 |
|---------|---------|------|------|
| **内参** | 焦距 | fx, fy | 2个 |
| | 主点坐标 | cx, cy | 2个 |
| | 畸变系数 | k1, k2, p1, p2, k3 | 5个 |
| **外参** | 旋转矩阵 | R | 3×3 |
| | 平移向量 | t | 3×1 |

### 1.3 为什么需要相机标定？

- **去畸变**：消除镜头畸变，使图像中的直线保持直线
- **视觉测量**：从图像中恢复物体的真实尺寸
- **三维重建**：从多个视角恢复场景的三维结构
- **机器人导航**：精确定位和物体识别

### 1.4 标定方法对比

| 方法 | 精度 | 设备要求 | 复杂度 |
|-----|------|---------|--------|
| 传统标定法 | 高 | 已知标定物 | 中 |
| 张正友标定法 | 高 | 棋盘格 | 低 |
| 自标定 | 低 | 无 | 高 |

本程序使用**张正友标定法**，这是目前最广泛使用的相机标定方法。

## 2. 张正友标定法原理

### 2.1 核心思想

张正友标定法的核心思想是：

1. **已知图案**：使用打印的棋盘格图案（世界坐标已知）
2. **多视角拍摄**：从不同角度拍摄棋盘格
3. **单应性矩阵**：从每个视角计算单应性矩阵
4. **参数求解**：从多个单应性矩阵中分解出内参和外参
5. **非线性优化**：通过最小化重投影误差优化所有参数

### 2.2 数学模型

#### 2.2.1 针孔相机模型

```
                        ┌─────────────────────────┐
                        │         场景             │
                        │                          │
                        │        (X, Y, Z)        │
                        │            │             │
                        │            │             │
                        │            ▼             │
                        │      ┌─────────┐         │
                        │      │  光心   │         │
                        │      │   O     │         │
                        │      └────┬────┘         │
                        │           │              │
                        │           │              │
                        │           ▼              │
                        │    ┌─────────────┐       │
                        │    │   图像平面   │       │
                        │    │   (u, v)    │       │
                        │    └─────────────┘       │
                        └─────────────────────────┘
```

#### 2.2.2 投影公式

从世界坐标到图像坐标的变换：

$$\begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = K \cdot \begin{bmatrix} R & t \end{bmatrix} \cdot \begin{bmatrix} X_w \\ Y_w \\ Z_w \\ 1 \end{bmatrix}$$

其中：
- $K$ 是相机内参矩阵
- $R$ 是旋转矩阵
- $t$ 是平移向量

#### 2.2.3 内参矩阵形式

$$K = \begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}$$

#### 2.2.4 畸变模型

**径向畸变**：
$$x_{distorted} = x(1 + k_1 r^2 + k_2 r^4 + k_3 r^6)$$
$$y_{distorted} = y(1 + k_1 r^2 + k_2 r^4 + k_3 r^6)$$

**切向畸变**：
$$x_{distorted} = x + 2p_1 xy + p_2(r^2 + 2x^2)$$
$$y_{distorted} = y + p_1(r^2 + 2y^2) + 2p_2 xy$$

其中 $r^2 = x^2 + y^2$

## 4. 第一步：加载标定图像

### 4.1 功能说明

从文件系统中读取标定图像，存储为OpenCV的Mat对象。

### 4.2 代码实现

```cpp
// 文件: camera_calibration.cpp
// 行号: 261-300

// 1. 配置图像路径
vector<string> imagePaths;

// 方式1：从命令行参数读取
if (argc > 1) {
    for (int i = 1; i < argc; i++) {
        imagePaths.push_back(argv[i]);
    }
}
// 方式2：从默认目录读取
else {
    cout << "用法示例：" << endl;
    cout << "  " << argv[0] << " image1.jpg image2.jpg ..." << endl;
    
    // 默认从calibration_images目录读取
    for (int i = 1; i <= 20; i++) {
        string path = "calibration_images/image_" + to_string(i) + ".jpg";
        imagePaths.push_back(path);
    }
}

// 2. 加载图像
vector<Mat> images;
cout << "\n正在加载标定图像..." << endl;

for (const string& path : imagePaths) {
    Mat img = imread(path);  // imread函数读取图像
    if (!img.empty()) {      // 检查图像是否加载成功
        images.push_back(img);
        cout << "  已加载: " << path << endl;
    }
}

// 3. 验证图像
if (images.empty()) {
    cerr << "\n错误：未能加载任何图像！" << endl;
    return -1;
}

Size imageSize = images[0].size();  // 获取图像尺寸
```

### 4.3 关键函数详解

#### 4.3.1 `imread()`


| 参数 | 说明 |
|-----|------|
| filename | 图像文件路径 |
| flags | 加载标志 |

1. **图像格式**：支持常见格式（jpg, png, bmp等）
2. **图像质量**：建议使用高分辨率、清晰的图像
3. **数量要求**：至少需要3张成功检测的图像，建议10-20张
4. **图像尺寸**：所有图像应尺寸一致

## 5. 第二步：检测棋盘角点

### 5.1 功能说明

在每张标定图像中检测棋盘格的内角点位置，这是标定过程中最关键的步骤。

### 5.2 代码实现

```cpp
// 文件: camera_calibration.cpp
// 行号: 22-85

int findChessboardCorners(
    const vector<Mat>& images,
    Size boardSize,
    vector<vector<Point2f>>& imagePoints,
    bool showResults = true)
{
    int successCount = 0;
    
    for (size_t i = 0; i < images.size(); i++) {
        // 1. 转换为灰度图
        Mat gray;
        cvtColor(images[i], gray, COLOR_BGR2GRAY);
        
        // 2. 检测角点
        vector<Point2f> corners;
        bool found = cv::findChessboardCorners(
            gray,
            boardSize,
            corners,
            CALIB_CB_ADAPTIVE_THRESH | CALIB_CB_NORMALIZE_IMAGE | CALIB_CB_FAST_CHECK
        );
        
        // 3. 亚像素级优化
        if (found) {
            cornerSubPix(
                gray,
                corners,
                Size(5, 5),              // 搜索窗口大小
                Size(-1, -1),            // 死区大小
                TermCriteria(TermCriteria::EPS + TermCriteria::COUNT, 30, 0.001)
            );
            
            imagePoints.push_back(corners);
            successCount++;
            
            // 4. 可视化
            if (showResults) {
                Mat imageWithCorners = images[i].clone();
                drawChessboardCorners(imageWithCorners, boardSize, corners, found);
                
                Mat display;
                resize(imageWithCorners, display, Size(), 0.5, 0.5);
                imshow("检测到的角点", display);
                waitKey(500);
            }
        }
    }
    
    if (showResults) {
        destroyWindow("检测到的角点");
    }
    
    return successCount;
}
```

### 5.3 关键函数详解

#### 5.3.1 `cvtColor()`

```cpp
void cvtColor(InputArray src, OutputArray dst, int code, int dstCn = 0);
```

**功能**：颜色空间转换

**常用代码**：
- `COLOR_BGR2GRAY`: BGR转灰度
- `COLOR_GRAY2BGR`: 灰度转BGR
- `COLOR_BGR2HSV`: BGR转HSV

**为什么需要灰度图？**
- 角点检测基于梯度信息，灰度图足够
- 减少计算量（单通道 vs 三通道）
- 消除颜色对检测的干扰

#### 5.3.2 `findChessboardCorners()`

```cpp
bool findChessboardCorners(
    InputArray image,
    Size patternSize,           // 棋盘内角点数量 (宽, 高)
    OutputArray corners,
    int flags = CALIB_CB_ADAPTIVE_THRESH
);
```

**功能**：检测棋盘格角点

**参数详解**：

| 标志 | 值 | 说明 |
|-----|---|------|
| `CALIB_CB_ADAPTIVE_THRESH` | 1 | 使用自适应阈值而非固定阈值 |
| `CALIB_CB_NORMALIZE_IMAGE` | 2 | 在阈值化前归一化图像对比度 |
| `CALIB_CB_FAST_CHECK` | 8 | 快速检查是否可能有角点 |
| `CALIB_CB_FILTER_QUADS` | 4 | 过滤掉非四边形的轮廓 |

**返回值**：
- `true`: 成功检测到所有角点
- `false`: 检测失败

**棋盘参数说明**：

```

│                                                             │

│  │                                                      │    │
│  │  ●  ●  ●  ●  ●  ●  ●  ●  ●                      │    │
```

**检测原理**：
1. 将灰度图转换为二值图（自适应阈值）
2. 寻找轮廓
3. 筛选四边形轮廓

#### 5.3.3 `cornerSubPix()`

```cpp
void cornerSubPix(
    InputArray image,
    InputOutputArray corners,
    Size winSize,           // 搜索窗口大小 (宽度, 高度)
    Size zeroZone,          // 死区大小 (-1表示不使用)
    TermCriteria criteria   // 迭代终止条件
);
```

**功能**：亚像素级角点精确化

**参数详解**：

| 参数 | 说明 | 常用值 |
|-----|------|-------|
| winSize | 搜索窗口半径（实际大小为2×winSize+1） | Size(5,5) → 11×11像素 |
| zeroZone | 死区半径，中心区域不搜索 | Size(-1,-1) → 不使用 |
| criteria | 迭代终止条件 | 见下文 |

**TermCriteria**：

```cpp
TermCriteria(
    int type,       // 终止类型
    int maxCount,   // 最大迭代次数
    double epsilon // 最小精度
);
```

**类型组合**：
- `TermCriteria::EPS`: 精度达到epsilon时停止
- `TermCriteria::COUNT`: 迭代次数达到maxCount时停止
- `TermCriteria::EPS + TermCriteria::COUNT`: 满足任一条件即停止

**优化原理**：

```
┌─────────────────────────────────────────────────────────────────┐
│              亚像素角点优化原理                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  初始估计                                                       │
│       ●                                                        │
│                                                                 │
│  迭代搜索（基于灰度梯度）                                        │
│       ↓  ↓  ↓                                                  │
│       ↓   ●    ↓    ← 在窗口内找到梯度方向交点                  │
│       ↓  ↓  ↓                                                  │
│                                                                 │
│  最终结果                                                       │
│         ★                                                       │
│         ↑                                                       │
│         精确的亚像素位置                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**为什么需要亚像素精度？**
- 像素级精度：位置精度为1像素
- 亚像素精度：位置精度可达0.01像素或更高
- 标定对精度要求高，亚像素优化必不可少

**功能**：在图像上绘制检测到的角点

**绘制效果**：

**功能**：

### 5.4 角点检测失败的可能原因

| 原因 | 解决方案 |
|-----|---------|
| 棋盘尺寸设置错误 | 检查boardSize参数 |

## 6. 第三步：生成世界坐标点

### 6.1 功能说明

生成棋盘格角点在世界坐标系中的三维坐标。因为棋盘是平面的，所有角点的Z坐标都为0。

### 6.2 代码实现

```cpp
// 文件: camera_calibration.cpp
// 行号: 93-109

void generateObjectPoints(
    Size boardSize,
    float squareSize,
    vector<Point3f>& objectPoints)
{
    objectPoints.clear();
    
    // 遍历棋盘的所有内角点
    for (int i = 0; i < boardSize.height; i++) {
        for (int j = 0; j < boardSize.width; j++) {
            // (X, Y, Z) = (j×squareSize, i×squareSize, 0)
            objectPoints.push_back(
                Point3f(j * squareSize, i * squareSize, 0)
            );
        }
    }
}

// 在main函数中调用
vector<Point3f> objectPointsPattern;
generateObjectPoints(boardSize, squareSize, objectPointsPattern);

// 为每张图像复制相同的世界坐标点
vector<vector<Point3f>> objectPoints(imagePoints.size(), objectPointsPattern);
```

#### 7.3.2 标定算法流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       cv::calibrateCamera内部流程                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  步骤1：初始化                                                          │
│  ├── 假设没有畸变 (k1=k2=k3=p1=p2=0)                                   │
│  ├── 假设主点在图像中心                                                 │
│  └── 初始焦距从图像尺寸估计                                             │
│                                                                         │
│  步骤2：单应性矩阵估计（每张图像）                                       │
│  ├── 从世界坐标→图像坐标的映射                                          │
│  ├── 使用直接线性变换(DLT)或RANSAC                                     │
│  └── 分解单应性矩阵得到外参初值                                         │
│                                                                         │
│  步骤3：内参约束方程                                                    │
│  ├── 从所有图像的单应性矩阵中提取约束                                   │
│  └── 求解内参矩阵的封闭解                                               │
│                                                                         │
│  步骤4：外参计算                                                        │
│  ├── 使用已知的内参计算每张图像的外参                                   │
│  ├── 旋转向量→旋转矩阵（Rodrigues变换）                                 │
│  └── 得到rvecs和tvecs                                                  │
│                                                                         │
│  步骤5：非线性优化（LM/Levenberg-Marquardt）                            │
│  ├── 最小化重投影误差                                                   │
│  ├── 优化所有参数：内参 + 外参 + 畸变系数                               │
│  └── 迭代直到收敛或达到最大迭代次数                                     │
│                                                                         │
│  步骤6：计算重投影误差                                                  │
│  ├── 用优化后的参数重新投影所有点                                       │
│  ├── 计算投影点与实际点的距离                                           │
│  └── 计算RMS值                                                          │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 10. 核心数学原理详解

### 10.1 坐标系转换

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          坐标系转换链条                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  世界坐标 (Xw, Yw, Zw)                                                  │
│       │                                                                 │
│       │  [R | t]                                                        │
│       │  刚体变换                                                        │
│       ▼                                                                 │
│  相机坐标 (Xc, Yc, Zc)                                                  │
│       │                                                                 │
│       │  相机内参矩阵K                                                   │
│       │  [fx  0  cx]                                                    │
│       │  [0  fy  cy]                                                    │
│       │  [0   0   1 ]                                                   │
│       ▼                                                                 │
│  归一化相机坐标 (x, y)                                                   │
│       │                                                                 │
│       │  畸变校正                                                        │
│       ▼                                                                 │
│  像素坐标 (u, v)                                                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 10.2 投影公式详细推导

**步骤1：世界坐标 → 相机坐标**

$$\begin{bmatrix} X_c \\ Y_c \\ Z_c \end{bmatrix} = R \cdot \begin{bmatrix} X_w \\ Y_w \\ Z_w \end{bmatrix} + t$$

**步骤2：相机坐标 → 归一化坐标**

$$\begin{bmatrix} x \\ y \\ 1 \end{bmatrix} = \frac{1}{Z_c} \cdot \begin{bmatrix} X_c \\ Y_c \\ Z_c \end{bmatrix} = K^{-1} \cdot \begin{bmatrix} X_c \\ Y_c \\ Z_c \\ 1 \end{bmatrix}$$

**步骤3：畸变校正**

$$x_{corrected} = x(1 + k_1 r^2 + k_2 r^4 + k_3 r^6) + 2p_1 xy + p_2(r^2 + 2x^2)$$
$$y_{corrected} = y(1 + k_1 r^2 + k_2 r^4 + k_3 r^6) + p_1(r^2 + 2y^2) + 2p_2 xy$$

其中 $r^2 = x^2 + y^2$

**步骤4：像素坐标**

$$u = f_x \cdot x_{corrected} + c_x$$
$$v = f_y \cdot y_{corrected} + c_y$$

### 10.3 重投影误差计算

对于每个角点：

1. **已知**：世界坐标 $P_w = (X, Y, Z)$
2. **投影**：使用标定参数计算 $P_{proj} = (u', v')$
3. **比较**：计算与实际检测点 $P_{detect} = (u, v)$ 的距离

$$d_i = \sqrt{(u_i' - u_i)^2 + (v_i' - v_i)^2}$$

**RMS计算**：

$$RMS = \sqrt{\frac{1}{N} \sum_{i=1}^{N} d_i^2}$$

其中 $N$ 是所有图像中所有角点的总数。

### 10.4 优化目标函数

标定过程实际上是求解以下最小二乘问题：

$$\min_{K, R_i, t_i, d} \sum_{i=1}^{M} \sum_{j=1}^{N} \| p_{ij} - \pi(K, d, R_i, t_i, P_{ij}) \|^2$$

其中：
- $M$: 图像数量
- $N$: 每张图像的角点数量
- $p_{ij}$: 第i张图像第j个角点的实际位置
- $\pi(\cdot)$: 投影函数
- $d$: 畸变系数

### 10.5 旋转的表示方法

#### 10.5.1 旋转矩阵

3×3正交矩阵，满足 $R^T R = I$ 且 $\det(R) = 1$

```cpp
// 示例：绕X轴旋转30度
Mat R = (Mat_<double>(3,3) << 
    1, 0, 0,
    0, 0.866, -0.5,
    0, 0.5, 0.866);
```

#### 10.5.2 旋转向量

使用罗德里格斯旋转公式：

$$R = I + \sin(\theta) \cdot K + (1 - \cos(\theta)) \cdot K^2$$

其中 $K$ 是旋转向量 $r = (r_x, r_y, r_z)$ 构造的反对称矩阵。

```cpp
// 旋转向量 → 旋转矩阵
Mat R;
Rodrigues(rvec, R);

// 旋转矩阵 → 旋转向量
Rodrigues(R, rvec);
```

```
旋转向量                    旋转矩阵
┌─────────┐              ┌─────────┐
│  [rx]   │              │         │
│  [ry]   │  ─Rodrigues─▶│    R    │
│  [rz]   │              │         │
└─────────┘              └─────────┘

• 方向：旋转轴
• 模长：旋转角度（弧度）
```

## 11. 常见问题与调试

### 11.1 角点检测失败

**问题**：无法检测到棋盘角点

**排查步骤**：

```
┌─────────────────────────────────────────────────────────────────┐
│              角点检测失败排查流程                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 检查棋盘参数                                                 │
│     └── boardSize是否正确？（内角点数量）                        │
│                                                                 │
│  2. 检查图像质量                                                 │
│     ├── 图像是否清晰？                                          │
│     ├── 棋盘是否完整？                                          │
│     └── 光照是否均匀？                                          │
│                                                                 │
│  3. 尝试调试参数                                                 │
│     ├── 添加CALIB_CB_NORMALIZE_IMAGE                           │
│     ├── 添加CALIB_CB_FAST_CHECK                                │
│     └── 降低自适应阈值块大小                                    │
│                                                                 │
│  4. 手动检查                                                     │
│     └── 用图像查看器确认棋盘清晰可见                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 11.2 重投影误差过大

**问题**：RMS > 2.0 像素

**解决方案**：

| 原因 | 解决方法 |
|-----|---------|
| 图像数量不足 | 增加至15-20张 |
| 图像覆盖不全 | 确保覆盖各个角度和位置 |
| 棋盘不平整 | 使用硬板粘贴棋盘 |
| 图像模糊 | 重新拍摄清晰图像 |
| 角点检测不准确 | 检查亚像素优化参数 |

### 11.3 畸变参数异常

**问题**：k1值过大或为NaN

**排查**：
1. 确保至少有3张成功检测的图像
2. 尝试固定部分畸变参数：`CALIB_FIX_K1 | CALIB_FIX_K2`
3. 检查棋盘尺寸参数是否正确

### 11.5 标定质量评估

**定量评估**：

| 指标 | 良好值 | 需改进值 |
|-----|-------|---------|
| RMS | < 1.0 | > 2.0 |
| k1绝对值 | < 0.5 | > 1.0 |
| 角点检测率 | > 80% | < 50% |

**定性评估**：
- 去畸变后直线应保持直线
- 棋盘格方格应为正方形
- 图像边缘无明显拉伸或压缩
